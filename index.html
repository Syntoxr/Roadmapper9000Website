Roadmapper 9000

Projektziel

Entwicklung einer Anwendung zur Bewertung, Protokollierung und Übermittlung der Wegebeschaffenheit von Fahrrad-Routen. Verwendetes Gerät: M5Stack Core 2. Datenerfassung durch Erschütterungsmessung auf verschiedenen Wegtypen.

Klassifizierung der Wegebeschaffenheit

Wegtyp Bewertung
Glatter Asphalt 100
Pflaster 80
Kies-/Schotterweg 50
Feldweg 30
Kopfsteinpflaster 20
Bordsteinkante 10
Tiefes Schlagloch 0

Umsetzungsplan

Meilenstein Umsetzung
Datenerfassung Rohdaten von Sensoren sammeln
Analyse Verwendung der KNIME Analytics Platform
Klassifizierung Strategie basierend auf den Daten entwickeln

Datenerfassung

  • Rohdaten von Sensoren sammeln
  • Rohdaten auf SD-Karte speichern
  • Für verschiedene Untergründe wiederholen

Datenanalyse

  • Verwendung der KNIME Analytics Platform
  • Verarbeitungstests durchführen
  • Funktionalität bestimmen

Klassifizierungsstrategie

  • Schwellenwerte für jeden Wegetyp basierend auf Testdaten definieren
  • Echtzeitvergleich der Sensordaten mit den Schwellenwerten

Datenübermittlung

  • Startnachricht an den Server senden
  • JSON-Nachricht via HTTP-POST senden
  • Aufnahme nach 5 Minuten Inaktivität stoppen

VS-Code für die Entwicklung

  • Erweiterbar mit Plugins
    • PlatformIO
    • Git Integration

Programmstruktur

Tasks

  • NtpTask
  • DisplayTask
  • GpsTask
  • ImuTask
  • RoadmapperTask

Vorteil FreeRTOS: Tasks laufen weitestgehend unabhängig von einander

Wichtige Funktionen

  • NtpTask: Zeit via Wi-Fi synchronisieren
  • ImuTask: IMU-Datenerhebung und verarbeitung
  • DisplayTask: Display mit GPS-Daten und Wegtyp aktualisieren
  • GpsTask: GPS-Daten sammeln
  • RoadmapperTask: Datenübertragung

NtpTask: Zeit via Wi-Fi synchronisieren. DisplayTask: Display mit GPS-Daten und Wegtyp aktualisieren. GpsTask: GPS-Daten sammeln.

Klassifizierung & Ereigniserkennung

  • Kontinuierliche Analyse für Wegtyp
  • Effiziente Berechnung mittels Exponential Moving Average
  • Schlaglöcher identifizieren durch Anomalien
